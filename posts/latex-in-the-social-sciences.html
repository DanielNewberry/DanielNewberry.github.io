LaTeX in the social sciences

2024-06-13

LaTeX is a software system for document preparation. When using it, we write in plain text as opposed to the instantly formatted text found in regular word processors like Microsoft Word and Apple Pages. LaTeX is generally commonplace in disciplines such as engineering, computer science and mathematics due to the superior equation and algorithm formatting than is found in other software systems. However, we are starting to see a gradual increase in the use of LaTeX in other areas, such as the social sciences.

To start, here is an example of some LaTeX plain text that we use:

\documentclass[12pt]{article}
\usepackage{amsmath}
\title{\LaTeX}
\date{}
\begin{document}
  \maketitle
  \LaTeX{} is a document preparation system for the \TeX{}
  typesetting program. It offers programmable desktop publishing
  features and extensive facilities for automating most aspects of
  typesetting and desktop publishing, including numbering and
  cross-referencing, tables and figures, page layout, bibliographies,
  and much more. \LaTeX{} was originally written in 1984 by Leslie
  Lamport and has become the dominant method for using \TeX; few
  people write in plain \TeX{} anymore. The current version  is
  \LaTeXe.

  % This is a comment; it will not be shown in the final output.
  % The following shows a little of the typesetting power of LaTeX:
  \begin{align}
    E &= mc^2
\\
    m &= \frac{m_0}{\sqrt{1-\frac{v^2}{c^2}}}
  \end{align}
\end{document}

Benefits

While itâ€™s true that LaTeX has its origins in mathematics and computer science, and that a large part of the userbase are from technical fields, many of the advantages offered by LaTeX are just as relevant for those within the social sciences and other disciplines.

Separation of style and content

LaTex allows the content and the style of the document to be two separate elements. This approach makes it easy to reuse designs and content without them being dependent on each other. For example, there are countless occasions where I have copied text from a Word file and had to spend a lot of time reformatting it to fit in with the style of the new document.

By copying the input text across to the new document and compiling with a new theme applied, all reformatting issues are dealt with automatically.

Consistency

LaTeX handles most textual and graphical elements consistently throughout the document. Word processors also have the capability of producing consistent documents, however many people don't know how to make best use of this. When collaborating, we often end up with documents with multiple fonts, sizes and styles.

With LaTeX, this is kept out of our hands and kept consistent.

Stability

While there are features in word processors that make it possible to modularise documents, this is often neglected by many either out of difficulty to use or just not knowing the features are available. When these documents grow into larger theses and reports, they can often become unstable and the software can demand more resources from the computer to run. With multiple pieces of software open at once such as web browser, music streaming and reference manager (a classic combo of mine), we can begin to experience severe lag.

Plain text files can be read and edited by practically any computer without issue. The only real intensive part comes with compiling which whilst can take a couple of minutes for larger documents, is often no where near as intensive as using word processing software in the long (and short) term.

Issues (and how to get around them)

I'm not a programmer...

Neither am I. At first LaTeX can seem daunting, however, I think this is often the result of focusing on the syntax too much and not on the workflow. The difference between learning the syntax needed for LaTeX and learning how to use point-and-click software is arbitrary. One may seem more natural only because we're more familiar with it.

The real difference is between the workflows. For example LaTeX forces you to make decisions about structure, layout and design, but offers consistency. Word processors typically make no such demands, but it is up to you (and collaborators) to make sure that designs are consistent.

No one I know uses LaTeX...

This is a valid concern and was one of the major reasons why it took me so long to start learning and using LaTeX. It seemed easier to just stick with a word processor like Microsoft Word. And then I began working with someone who uses Apple Pages, and again with someone who uses Libre Office. Before I knew it, we were sharing PDFs between each other as this was the only middle ground that we could all use.

At this point, when I realised that sharing PDFs and commenting was the best way for us, I decided to start learning how to use LaTeX properly and share the compiled PDFs that way. Of course the issue is that, because (unfortunately) we have made Microsoft Word the industry standard, there are still people who can't or won't edit a document unless it is provided in a Word format. Whilst it is not a perfect solution, I would then use a service such as SmallPDF to convert the PDF file to a Word document. Not perfect, but it's something.

There are publishing issues...

Whilst yes, there are many journals that would not be able to accept a LaTeX file in its plain text form, most journals accept a PDF for submission. In which case, it doesn't really matter how you create the PDF, as long you create one. Word, Apple Pages, Libre Office, LaTeX, as long as you export to a PDF format for submission this should be fine.

And I would argue against subnmitting in any format other than PDF as there are be inconsistencies and formatting issues without the PDF standardisation.

I hope I've provided a solid argument for why LaTeX can be useful to know, even if it is not the main way that you write documents. A common alternative is to write the documents as you go in a standard word editor, before using LaTeX to create the final, designed version.

And of course, the more people from across different disciplines use LaTeX and the more common it becomes, the easier it will be to use in a collaborative way!
